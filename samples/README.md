# Sample logs, metrics, and traces

This directory contains one sample document per AWS service produced by the [AWS → Elastic Load Generator](../README.md).

| Directory    | Description                                                                                                                                                               |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **logs/**    | One sample log document per service (211 services). Each file is a single JSON document as generated by the app (nulls stripped).                                         |
| **metrics/** | One sample metrics document per metrics-supported service (189 services). Includes `data_stream.type: "metrics"`, `metricset`, and the appropriate `data_stream.dataset`. |
| **traces/**  | One sample APM trace document per trace-supported service (23 services). First span from each trace generator — transaction root or leading span.                         |

## Regenerating samples

From the project root:

```bash
npm run samples
```

This uses the same generators as the app and writes into `samples/logs/`, `samples/metrics/`, and `samples/traces/`. Use these samples as reference for index mapping, ingest pipelines, or testing ingestion.

## Verifying coverage

After changing generators, run:

```bash
npm run samples:verify
```

This fails with a non-zero exit code if any log, metric, or trace sample file is missing for a registered generator (or if unexpected files exist). Run `npm run samples` first to regenerate, then `npm run samples:verify` to confirm **211 + 189 + 23** files line up with `GENERATORS`, `METRICS_GENERATORS`, and `TRACE_GENERATORS`.
