# Sample logs, metrics, and traces

This directory contains one sample document per AWS service produced by the [AWS → Elastic Load Generator](../README.md).

| Directory    | Description |
|-------------|-------------|
| **logs/**   | One sample log document per service (165 services). Each file is a single JSON document as generated by the app (nulls stripped). |
| **metrics/** | One sample metrics document per metrics-supported service (139 services). Includes `data_stream.type: "metrics"`, `metricset`, and the appropriate `data_stream.dataset`. |
| **traces/**  | One sample APM trace document per trace-supported service (20 services). First span from each trace generator — transaction root or leading span. |

## Regenerating samples

From the project root:

```bash
npm run samples
```

This uses the same generators as the app and writes into `samples/logs/`, `samples/metrics/`, and `samples/traces/`. Use these samples as reference for index mapping, ingest pipelines, or testing ingestion.
