# Sample logs and metrics

This directory contains one sample document per AWS service produced by the [AWS → Elastic Load Generator](../README.md).

| Directory   | Description |
|------------|-------------|
| **logs/**  | One sample log document per service (134 services). Each file is a single JSON document as generated by the app (nulls stripped). |
| **metrics/** | One sample metrics document per service that supports metrics in the Elastic AWS integration (24 services). Same payload as logs plus `data_stream.type: "metrics"`, `metricset`, and the appropriate `data_stream.dataset`. |

## Regenerating samples

From the project root:

```bash
npm run samples
```

This uses the same generators as the app and writes into `samples/logs/` and `samples/metrics/`. Use these samples as reference for index mapping, ingest pipelines, or testing ingestion.
