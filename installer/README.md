# AWS → Elastic Onboarding Installers

Two standalone Node.js scripts to configure Elastic for use with the AWS → Elastic Load Generator.
Run them once before you start shipping data.

> **Requirements:** Node.js 18+ (uses native `fetch` and ES modules). No `npm install` needed — zero external dependencies.

---

## Installer 1 — Elastic AWS Integration (`elastic-integration`)

Installs the **official Elastic AWS integration package** via the Kibana Fleet API.

This gives you:
- Pre-built index templates for all 46 AWS services with native Elastic integration support
- ILM (Index Lifecycle Management) policies
- Pre-built Kibana dashboards for CloudTrail, VPC Flow, ALB/NLB, GuardDuty, Lambda, RDS, etc.
- ML job configurations for anomaly detection

### Run

```bash
npm run setup:integration
# or directly:
node installer/elastic-integration/index.mjs
```

### What it does

1. Prompts for your **Kibana URL** and **API key**
2. Checks if the AWS integration is already installed (`GET /api/fleet/epm/packages/aws`)
3. If already installed → skips (idempotent, safe to re-run)
4. If not installed → fetches the latest version from the Elastic Package Registry and installs it

### Credentials needed

| Prompt | Where to find it |
|--------|-----------------|
| Kibana URL | Stack Management → Integrations (copy the base URL, e.g. `https://my.kb.us-east-1.aws.elastic-cloud.com:9243`) |
| API key | Stack Management → API Keys → Create API key (needs `cluster: manage` + `kibana: all` privileges) |

## Installer 2 — Custom Ingest Pipelines (`custom-pipelines`)

Installs **custom Elasticsearch ingest pipelines** for the ~85 AWS services not covered by the official Elastic integration.

These pipelines parse the JSON `message` field emitted by the load generator into structured `*.parsed` fields, making logs fully searchable and dashboardable.

### Run

```bash
npm run setup:pipelines
# or directly:
node installer/custom-pipelines/index.mjs
```

### What it does

1. Prompts for your **Elasticsearch URL** and **API key**
2. Tests the connection and prints the cluster name
3. Shows a **group selection menu** — choose which service groups to install pipelines for:
   ```
   Select pipeline groups to install:
     1. analytics   (Glue, EMR, Athena)
     2. databases   (RDS, OpenSearch)
     3. serverless  (Lambda)
     4. compute     (EC2)
     5. management  (CloudFormation, SSM, CodeBuild, CodePipeline)
     6. iot         (IoT Core, Greengrass, SageMaker)
     7. all
   ```
4. For each selected pipeline:
   - Checks if already installed (`GET /_ingest/pipeline/{id}`)
   - Already installed → skips (idempotent)
   - Not installed → creates it (`PUT /_ingest/pipeline/{id}`)
5. Prints a summary of installed / skipped / failed pipelines

### Credentials needed

| Prompt | Where to find it |
|--------|-----------------|
| Elasticsearch URL | Deployment overview → Copy endpoint (e.g. `https://my.es.us-east-1.aws.elastic-cloud.com:9243`) |
| API key | Stack Management → API Keys → Create API key (needs `manage_ingest_pipelines` cluster privilege) |

---

## Pipeline naming convention

Custom pipelines follow the standard Elastic naming convention:

```
logs-aws.{dataset}-default
```

Examples:
- `logs-aws.glue-default` — parses Glue continuous logging JSON
- `logs-aws.lambda_logs-default` — handles Lambda START/END/REPORT log events
- `logs-aws.emr-default` — parses EMR structured JSON messages

These match the index patterns the load generator writes to, so pipelines are applied automatically on ingest.

---

## Why two separate installers?

| | `elastic-integration` | `custom-pipelines` |
|---|---|---|
| **API** | Kibana Fleet API | Elasticsearch Ingest API |
| **Credentials** | Kibana URL + API key | Elasticsearch URL + API key |
| **Update cadence** | When Elastic releases new integration versions | When new services are added to the load generator |
| **Re-runnable** | Yes — skips if already installed | Yes — skips existing pipelines |

Running both installers gives you full coverage across all 136 services generated by the load generator.
